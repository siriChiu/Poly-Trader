> 發展路線與進度。架構見 [ARCHITECTURE.md](ARCHITECTURE.md)，需求見 [PRD.md](PRD.md)，問題見 [ISSUES.md](ISSUES.md)。

> 六帽會議決議 (2026-03-30)：從「能跑」到「有意義」

## 核心理念

**Validation-First**：先證明每個感官有預測力，再組合。
多感官量化系統的價值不在於收集數據，而在於回答：
> 「哪些感官的設計是有效的？哪些需要改進？」

---

## 路線圖

### Phase 8: Validation-First（本週）
**目標**：累積 100+ 數據點，計算 IC，淘汰無效感官

- [ ] 自動化數據收集（每小時執行 collector + preprocessor）
- [ ] 計算每個感官的 Information Coefficient (IC)
- [ ] 多感官儀表板：IC 條形圖 + 分位數勝率熱圖
- [ ] 自動標記：IC < 0.05 → ⚠️ 需重新設計
- [ ] ISSUES.md 記錄每個感官的 IC 結果

### Phase 9: 多感官融合引擎（下週）
**目標**：從 DummyPredictor 升級為 IC 動態加權模型

- [ ] 模型權重 ∝ |IC|（資訊量越大的感官權重越高）
- [ ] 自動排除 IC ≈ 0 的感官
- [ ] 可解釋性輸出：「本次預測中，X 感官貢獻最大」
- [ ] 回測驗證：IC 加權 vs 等權重的 Sharpe 對比

### Phase 10: 收益歸因系統（2 週後）
**目標**：追蹤每個感官對交易結果的貢獻

- [ ] 每筆交易的「虛擬 P&L」：如果只看 Eye，會做出什麼決策？
- [ ] 長期追蹤：哪些感官穩定獲利，哪些穩定虧損
- [ ] 儀表板：多感官收益歸因瀑布圖

### Phase 11: 市場環境分類器（3 週後）
**目標**：自動識別市場狀態，動態調整感官權重

- [ ] 市場分類：牛市 / 熊市 / 震盪
- [ ] 不同環境下，不同感官的最優權重
- [ ] 即時環境感知 + 權重切換
