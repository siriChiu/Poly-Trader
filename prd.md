# 產品需求文件 (PRD)：Project Five-Senses 加密貨幣量化引擎

## 1. 產品概述
Project Five-Senses 是一個輕量化、低基礎設施成本的加密貨幣自動化量化交易系統。系統透過收集多維度免費公開數據（眼、耳、鼻、舌、身），進行特徵工程正規化，並利用機器學習模型（XGBoost/隨機森林）進行非線性決策（意），最終輸出交易信號並執行自動化下單。

## 2. 目標受眾與使用場景
* **使用者：** 具備開發能力的個人量化交易員。
* **運行環境：** 需能於低功耗環境（如 Raspberry Pi 或平價 VPS）24/7 穩定運行。

## 3. 核心功能模組要求

### 3.1 數據採集 (Data Ingestion) - 「五感」
系統必須定時從以下免費來源抓取數據，並具備 API 速率限制 (Rate Limit) 的重試與容錯機制：
* **眼 (技術與流動性)：** 透過 Binance API 獲取 K 線與成交量；透過 Coinglass 獲取清算痛點。
* **耳 (總經與共識)：** 透過 Polymarket API 獲取特定總經事件機率；抓取 ETF 資金淨流入數據。
* **鼻 (衍生品氣味)：** 透過 Binance Futures API 獲取 BTC 資金費率 (Funding Rate) 與未平倉量 (OI)。
* **舌 (社群情緒)：** 獲取 Alternative.me 恐懼貪婪指數；獲取 Binance 散戶多空比。
* **身 (鏈上資金)：** 透過 DefiLlama API 獲取全網穩定幣總市值。

### 3.2 特徵工程 (Feature Engineering)
將所有原始數據轉換為 -1 到 1 之間的標準化特徵：
* 必須實作 Z-score 標準化、Min-Max 縮放、以及變動率 (ROC) 計算。
* 需處理缺失值 (NaN)，若關鍵 API 斷線，系統應進入「安全觀望」模式，暫停產出交易信號。

### 3.3 決策引擎 (Decision Engine) - 「意」
* 實作 XGBoost 或 Random Forest 分類模型。
* 模型輸入：過去 24 小時的五感特徵矩陣。
* 模型輸出：未來 24 小時價格上漲/下跌的機率 (0~1)。

### 3.4 執行與風險控制 (Execution & Risk)
* 透過 CCXT 庫連接交易所進行現貨或低槓桿合約下單。
* 必須實作硬止損機制 (例如：跌破建倉價 3% 立即市價平倉)。
* 需實作倉位管理模組，依據模型輸出的勝率信心指數決定下單金額。

## 4. 非功能性需求
* **穩定性：** 具備詳細的 Logging 機制，記錄每一次 API 呼叫、特徵轉換結果與模型預測分數。
* **模組化：** 各個「感官」的爬蟲必須獨立解耦，單一 API 失效不得導致整個主程式崩潰。
